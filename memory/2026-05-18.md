## Workflow (Builder Mode)
1. Inspect repo
2. Make smallest useful change
3. Run `npm install` if deps changed
4. Run `npm run build` when possible
5. Fix build errors
6. Update README.md, OPENCLAW_BUILD_LOG.md, DEPLOYMENT_STATUS.md
7. Commit the change
8. Push to main
9. Tell students to check published app URL

## Safety Rules
- Never commit secrets / .env files
- Never expose API keys (MiniMax, Gemini, OpenAI, GitHub, Vercel, Supabase)
- Never create NEXT_PUBLIC_* variables for private keys
- If AI needed at runtime → use server-side API route

## Pedagogical Rule
Always explain in non-technical language: what I'm doing, why it matters, what to check, what doesn't work, next step.
